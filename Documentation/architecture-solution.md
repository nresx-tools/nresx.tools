ADR 002: [Вибір технології для роботи з файлами \*.xlsx](adr-002-excel.md)

Проект "Звітність" має на меті вдосконалити процесс збору щоденних звітів від всіх підрозділів аеророзвідки для подальшого аналізу командуванням.

### 1. Архітектура:

Додаток "Звітність" розробляється як автономний застосунок для операційної системи Windows. 
Він призначений для роботи в офлайн-режимі та орієнтований на роботу на захищених ДСК-комп'ютерах.

Застосунок передбачає обмін даними із зовнішніми системами виключно за допомогою файлів у форматах \*.xlsx або \*.json. 
Це передбачає як експорт звітів за конкретну дату в файли обраного формату для подальшої передачі в систему вищестоящего підрозділу, 
так і імпорт відповідних звітів переданих підпорядкованими підрозділами аеророзвідки.

У зв'язку з обмеженням на зберігання будь-яких даних на локальній робочій станції між сессіями роботи застосунку 
локальна база даних має зберігатись у вигляді окремих файлів на зовнішньому захищеному носії.

### 2. Технології:

#### Платформа:

Для реалізації додатку "Звітність", вибрано платформу .NET та мову програмування C#. 
Цей вибір забезпечує високий рівень сумісності з операційною системою Windows і забезпечує ефективність та надійність роботи застосунку. 
Використання .NET дозволяє швидку розробку та забезпечує доступ до великої екосистеми бібліотек та інструментів.
Мова програмування C# надає високий рівень безпеки та простоти в розробці, що сприяє створенню стабільного та продуктивного додатку.

#### Технології Інтерфейсу Користувача (UI):

Для інтерфейсу користувача додатку "Звітність" обрана технологія Windows Presentation Foundation (WPF). 
Цей вибір базується на декількох ключових перевагах. 
По-перше, WPF є сумісною зі старими версіями Windows, що забезпечує широке охоплення робочих станцій у кінцевих користувачів.
По-друге, WPF є стабільною та надійнійною технологія. Попри застарілість, вона досі активно підтримується Microsoft, що робить її стабільним та довготривалим вибором.
Крім того, WPF сумісна з останніми версіями .NET, що сприяє використанню новітніх можливостей платформи. 
Додатковим фактором є наявність в команді розробників із відповідним досвідом, що полегшує роботу команди на ранніх етапах проекту.

#### База даних:

#### Технології роботи з файлами excel:

Для роботи з файлами \*.xlsx була обрана бібліотека [EPPlus 7.x](https://www.epplussoftware.com)
Ця бібліотека відповідає всім критеріям наших вимог, детальніше з крітеріями вибору можна ознайомитись в: 
[Рішення по вибору технології для роботи з файлами \*.excel](adr-002-excel.md)

### 3. Безпека:

Для автентифікація користувача буде використана система авторизації та автентифікації windows, додаткової імплементації в застосунку не передбачається.

Застосунок має працювати на захищених ДСК робочих станціях, та не має зберігати жодні дані користувача на робочій станції. 
Локальна БД, та згенеровані звіти мають зберігатись на зовнішніх захищених носіях.
